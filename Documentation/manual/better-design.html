Improving on the Game Design

1. Better Game Progression

One of the weaknesses of the game in its current state is the lack of build-up or progression in the game. Players are introduced to all the mechanics at once without in-depth explanation to any of them. Thus one simple (though not necessarily easy) improvement to make to the game is to improve the pace at which mechanics are introduced to the player.

This would probably require a script that can track the game progression, and toggle certain parts of the game on or off. It would also benefit from some UI elements and animations that can highlight certain parts of the game. For a starting point, one can look at how the TutorialBehaviour uses the Overlay Manager to highlight parts of the game that need attention.

2. Better Time Pressure

Another weakness of the game is that players have no reason to actually work under constraints. It is possible to finish the game without playing at all, just by letting the clock run out. This manner of playing the game would not trigger any story events, but does not have detriments otherwise. Perhaps a way to punish overly slow decision making can be devised and implemented.
