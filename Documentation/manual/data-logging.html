Data Logging

In order that the game might be used for research of some sort, data collected from the game has to be logged somewhere that can be accessed by the researcher. Thankfully, achieving the work required for a save system is already half the work needed for data logging.

Like saving and loading, data logging can make use of serialization to store data about the game. However, unlike saving and loading which can possibly be done on the player's computer locally, data collected from the game needs to be collated together, and be accessible in a central place that is quite possibly not the computer running the game.

Thus for data logging, an external data storage solution is required, which might look like option 3 for achieving persistent game data. To this end there are a few (non-exhaustive) options:

1. A Database Server

This would involve setting up a database like MySQL on a server that the game will interact with. This would take quite a bit of expertise, and would likely require a person to be in charge of the server. Researchers would then access information through said person. This option would be favourable if the game is to be scaled up massively.

Requirements:

  • A server
  • Knowledge of database setup, use, and management
  • Knowledge of how to interface Unity WebGL and the database

Cost: non-zero, dependant on scale

2. Cloud/online solutions

Online services like Google Firebase provide API for relational databases that the game can use to add and query information. Being not as technically demanding as demanding as maintaining a database, researchers may want to create their own account and link the game to their own personal database on firebase. However, some might still need assistance of someone more familiar with such services to help. This option would be favourable if the scale of the study is large enough to use a database, but small enough such that using an online service would not be too costly.

Requirements:

  • How to use API from the service
  • Knowledge of how to interface Unity WebGL to use the API
    Cost: could be free, depending on scale

3. Google Sheets API

An google sheet can be used as a database of sorts. While not as robust as an actual database, a worksheet is easily accessible to most researchers, even if they are not familiar with coding. With proper documentation, anyone should be able to change some code to link the game to their own google account. This makes this option favourable if the game is to be geared for smaller scale research.

Requirements:

  • A Google account with a developer profile
  • (For Coder implementing) How to use Google Sheets API
  • (For researcher linking the game) How to Authenticate/Link the Google account
  • Knowledge of how to interface Unity WebGL to use the API Cost: free
